# Cursor Toolchain Index

Inventory of every Cursor rule, skill, and subagent in this repo, with
the ISO 8601 UTC timestamp of each file's most recent meaningful edit.
The agent reads this at session start to see at a glance what has
changed in the instruction layer.

This file is **regenerated by `bash scripts/regenerate-toolchain.sh`**
from the YAML frontmatter of every rule/skill/subagent file. Do not
hand-edit the tables below — your edits will be overwritten on the next
regeneration. Edit the source files' frontmatter (`description`,
`lastUpdated`) instead, then re-run the regenerator.

## Discipline

Whenever you edit any file under `.cursor/rules/`, `.cursor/skills/`,
or `.cursor/agents/`:

1. Bump that file's `lastUpdated:` in its YAML frontmatter to the current
   ISO 8601 UTC timestamp. Use the system `date` command — never guess:

   ```bash
   date -u +%Y-%m-%dT%H:%M:%SZ
   ```

2. Run `bash scripts/regenerate-toolchain.sh` to refresh this file.
3. If you added a new rule/skill/subagent, also add a row to
   `uds-docs/app/(site)/cursor-workflows/CursorWorkflowsClient.tsx` so
   `audit-agent-docs-currency.sh` passes.

CI enforces this via `scripts/audit-toolchain-currency.sh`, which fails
if any rule/skill/subagent file's git mtime is newer than its
`lastUpdated:` value or its row in this file is missing/stale. See
`.cursor/rules/uds-rule-discipline.mdc` for the full rule.

## Sync check

Before relying on anything below, confirm the local repo is current with
`origin/main`. The `sessionStart` Cursor hook at
`.cursor/hooks/sync-check.sh` runs `git fetch origin main` and writes
`.cursor/state/sync-status.json` (canonical) on every session start.
The `Phase 1: Verify sync state` step in `uds-master-preflight.mdc` is
the manual fallback when the hook hasn't run.
