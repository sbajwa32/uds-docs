Pre-Flight Checklist

These rules must be followed before completing any task that modifies the UDS documentation site.

1. Site Changelog

uds-site-changelog.mdc
  • Bump SITE YYYY.MM.DD.N in index.html
  • Add/update entry in SITE_CHANGELOG array in app.js
  • Increment ?v=N cache params on CSS/JS script tags

2. Token-First CSS

uds-token-first-css.mdc
  • No hardcoded hex colors, rgb(), or rgba() — use var(--uds-color-*)
  • No hardcoded px for font-size, spacing, radius — use var(--uds-*) tokens
  • No hardcoded font-family — use var(--uds-font-family)
  • Shadows use var(--uds-shadow-*) tokens

3. Component Checklist

uds-component-checklist.mdc
  • CSS: :hover, :focus-visible, disabled states
  • JS: IIFE pattern, _udsInit guard, keyboard handlers
  • Registered in uds.js orchestrator
  • PLAYGROUNDS, IMPL_DATA, CODE_TAB_CONFIGS, STATUS, FIGMA_LINKS entries
  • CHANGELOG entry + JS_FUNC_TO_FILE + React/Vue behavior hooks
  • CSS + JS files added to download ZIP array

4. Sample Code Standards

uds-sample-code-standards.mdc
  • React: proper hooks, no unused setters, no literal {…}
  • Vue: Composition API with <script setup>
  • All generated code must be valid and copy-pasteable

5. Release Workflow

uds-release-workflow.mdc
  • If UDS CHANGELOG was modified, sync Figma release notes IMMEDIATELY
  • Rebuild Release Notes in both UDS Tokens and UDS Components files

What is UDS?

UDS (Urban Design System) is a CSS-first design system with semantic tokens and BEM-style components. It works with any stack — HTML, React, Vue, Angular, Svelte — no framework dependency.

For AI Agents

If you're an AI coding assistant, here's what you need:

  • Machine-readable context: Fetch ai-context.json for the complete token list, component catalog, and framework rules in structured JSON.
  • Cursor rule: Download uds-design-system.mdc and place it in .cursor/rules/ — it contains everything inline.
  • Install UDS: npm install uds-core then import 'uds-core/uds.css'
  • Icons: Material Symbols Outlined from Google Fonts CDN.

Getting Started

UDS (Urban Design System) is a vanilla-first design system. It ships as CSS + a small JavaScript file — no framework dependency. It works with any stack: plain HTML, React, Vue, Angular, Svelte, or anything else. Add the files to your project, then copy component markup from the docs.

How it works

download 1. Get the files

Download the UDS bundle below (CSS + JS), or copy the uds/ folder from the shared repository.

link 2. Add to your project

Link uds.css for styling and uds.js for component behavior. The JS orchestrator auto-loads per-component scripts (one .js file per interactive component).

content_copy 3. Copy & paste

Browse component pages, copy the markup from any example or playground, and paste it into your project. Done.

Download

The bundle contains all tokens (colors, spacing, typography), every component stylesheet, and the JS behavior file. No dependencies, no build step.

download Download UDS (.zip)

Or, if your team uses a shared repo, copy the uds/ folder directly into your project.

npm

Install UDS as an npm package for React, Vue, or any bundler-based project.

npm install uds-core
// In your entry file (main.jsx, main.js, etc.)
import 'uds-core/uds.css';

Works with Vite, Next.js, and Nuxt. React and Vue manage component state natively — no uds.js needed.

CDN

Add UDS to any page with two tags — no build step required.

<link rel="stylesheet" href="https://sbajwa32.github.io/uds-docs/uds/uds.css" />
<script src="https://sbajwa32.github.io/uds-docs/uds/uds.js"></script>

Pin to a specific version:

<link rel="stylesheet" href="https://sbajwa32.github.io/uds-docs/versions/0.2/uds/uds.css" />

Setup by Framework

UDS is plain CSS — the setup is the same idea everywhere. Add the stylesheet and the icon font, then use the class names.

Plain HTML

Add the icon font, the stylesheet, and the JS file. No build tools needed.

<head>
  <!-- Material Symbols icon font (required for icons) -->
  <link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Material+Symbols+Outlined:opsz,wght,FILL,GRAD@20..48,100..700,0..1,-50..200" />

  <!-- UDS stylesheet -->
  <link rel="stylesheet" href="uds/uds.css" />
</head>

<body>
  <!-- Use any UDS component — copy markup from the docs -->
  <button class="udc-button-primary">Save changes</button>

  <!-- UDS behaviors (auto-initialises tabs, validation, etc.) -->
  <script src="uds/uds.js"></script>
</body>

React (CRA / Vite / Next.js)

Copy the uds/ folder into your src/ directory. Import the CSS once. In React, you manage state with hooks — UDS provides the styling via CSS classes, and you write the logic natively.

/* src/index.js or src/main.jsx */
import './uds/uds.css';

import { useState } from 'react';

function SearchInput() {
  const [query, setQuery] = useState('');

  return (
    <div className="udc-text-input">
      <label className="udc-text-input__label">Search</label>
      <div className="udc-text-input__field">
        <span className="udc-text-input__leading-icon">
          <span className="material-symbols-outlined">search</span>
        </span>
        <input
          type="text"
          placeholder="Search..."
          value={query}
          onChange={(e) => setQuery(e.target.value)}
        />
      </div>
    </div>
  );
}

Note: In React you don't need uds.js — React handles component logic natively via hooks. The UDS CSS classes provide all the styling.

Vue 3

Copy the uds/ folder into your project. Import the CSS once. Vue handles state with ref() and v-model — UDS provides the styling.

/* main.js */
import './uds/uds.css';
import { createApp } from 'vue';
import App from './App.vue';
createApp(App).mount('#app');

<script setup>
import { ref } from 'vue';

const query = ref('');
</script>

<template>
  <div class="udc-text-input">
    <label class="udc-text-input__label">Search</label>
    <div class="udc-text-input__field">
      <span class="udc-text-input__leading-icon">
        <span class="material-symbols-outlined">search</span>
      </span>
      <input type="text" placeholder="Search..." v-model="query" />
    </div>
  </div>
</template>

Note: In Vue you don't need uds.js — Vue handles reactivity natively. The UDS CSS classes provide all the styling.

Angular / Svelte / Others

Same approach — import uds.css in your global styles and optionally add uds.js for vanilla component behavior. Each interactive component has its own .js file that uds.js auto-loads. The class names and HTML patterns are the same everywhere. Use the HTML / CSS tab on each component page for the base markup, and handle state in your framework's native way.

Theming

UDS supports multiple themes via data-* attributes on the <html> element. Set them to switch color schemes, brands, fonts, and density at runtime — no rebuild needed.

<!-- Light mode (default — no attributes needed) -->
<html>

<!-- Dark mode -->
<html data-color-scheme="dark">

<!-- Brand override -->
<html data-theme="resman">

<!-- Combine multiple -->
<html data-color-scheme="dark" data-theme="anyonehome" data-font="poppins">

When to use

  • Supplementary information on icon-only buttons
  • Brief explanations for UI elements
  • Showing full text for truncated content

Implementation

  • CSS-only — no JavaScript required
  • Shows on hover of the wrapper or focus-within
  • Max-width 250px, text wraps naturally
  • Shadow uses --uds-shadow-depth-300 token

Accessibility

  • Always include role="tooltip"
  • Link trigger to tooltip with aria-describedby for complex cases
  • Tooltip shows on focus-within, so keyboard users can access it
