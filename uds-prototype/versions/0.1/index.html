Getting Started

UDS (Urban Design System) is a vanilla-first design system. It ships as CSS + a small JavaScript file — no framework dependency. It works with any stack: plain HTML, React, Vue, Angular, Svelte, or anything else. Add the files to your project, then copy component markup from the docs.

How it works

download 1. Get the files

Download the UDS bundle below (CSS + JS), or copy the uds/ folder from the shared repository.

link 2. Add to your project

Link uds.css for styling and uds.js for component behavior (tabs, validation, etc.).

content_copy 3. Copy & paste

Browse component pages, copy the markup from any example or playground, and paste it into your project. Done.

Download

The bundle contains all tokens (colors, spacing, typography), every component stylesheet, and the JS behavior file. No dependencies, no build step.

download Download UDS (.zip)

Or, if your team uses a shared repo, copy the uds/ folder directly into your project.

Setup by Framework

UDS is plain CSS — the setup is the same idea everywhere. Add the stylesheet and the icon font, then use the class names.

Plain HTML

Add the icon font, the stylesheet, and the JS file. No build tools needed.

<head>
  <!-- Material Symbols icon font (required for icons) -->
  <link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Material+Symbols+Outlined:opsz,wght,FILL,GRAD@20..48,100..700,0..1,-50..200" />

  <!-- UDS stylesheet -->
  <link rel="stylesheet" href="uds/uds.css" />
</head>

<body>
  <!-- Use any UDS component — copy markup from the docs -->
  <button class="udc-button-primary">Save changes</button>

  <!-- UDS behaviors (auto-initialises tabs, validation, etc.) -->
  <script src="uds/uds.js"></script>
</body>

React (CRA / Vite / Next.js)

Copy the uds/ folder into your src/ directory. Import the CSS once. In React, you manage state with hooks — UDS provides the styling via CSS classes, and you write the logic natively.

/* src/index.js or src/main.jsx */
import './uds/uds.css';

import { useState } from 'react';

function SearchInput() {
  const [query, setQuery] = useState('');

  return (
    <div className="udc-text-input">
      <label className="udc-text-input__label">Search</label>
      <div className="udc-text-input__field">
        <span className="udc-text-input__leading-icon">
          <span className="material-symbols-outlined">search</span>
        </span>
        <input
          type="text"
          placeholder="Search..."
          value={query}
          onChange={(e) => setQuery(e.target.value)}
        />
      </div>
    </div>
  );
}

Note: In React you don't need uds.js — React handles component logic natively via hooks. The UDS CSS classes provide all the styling.

Vue 3

Copy the uds/ folder into your project. Import the CSS once. Vue handles state with ref() and v-model — UDS provides the styling.

/* main.js */
import './uds/uds.css';
import { createApp } from 'vue';
import App from './App.vue';
createApp(App).mount('#app');

<script setup>
import { ref } from 'vue';

const query = ref('');
</script>

<template>
  <div class="udc-text-input">
    <label class="udc-text-input__label">Search</label>
    <div class="udc-text-input__field">
      <span class="udc-text-input__leading-icon">
        <span class="material-symbols-outlined">search</span>
      </span>
      <input type="text" placeholder="Search..." v-model="query" />
    </div>
  </div>
</template>

Note: In Vue you don't need uds.js — Vue handles reactivity natively. The UDS CSS classes provide all the styling.

Angular / Svelte / Others

Same approach — import uds.css in your global styles and add uds.js for component behavior. The class names and HTML patterns are the same everywhere. Use the HTML / CSS tab on each component page for the base markup, and handle state in your framework's native way.

Theming

UDS supports multiple themes via data-* attributes on the <html> element. Set them to switch color schemes, brands, fonts, and density at runtime — no rebuild needed.

<!-- Light mode (default — no attributes needed) -->
<html>

<!-- Dark mode -->
<html data-color-scheme="dark">

<!-- Brand override -->
<html data-theme="resman">

<!-- Combine multiple -->
<html data-color-scheme="dark" data-theme="anyonehome" data-font="poppins">

Available theme attributes

AttributeValuesEffect
data-color-schemedarkDark mode (light is default)
data-themeresman, anyonehome, inhabitBrand accent colors
data-fontpoppins, roboto, lexendFont family override
data-font-scalesmaller, largerGlobal font size scale
data-densitycomfortableRelaxed spacing

Try these live using the appearance bar at the top of this site.
