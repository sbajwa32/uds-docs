Pre-Flight Checklist

These rules must be followed before completing any task that modifies the UDS documentation site.

1. Site Changelog

uds-site-changelog.mdc
  • Bump SITE YYYY.MM.DD.N in index.html
  • Add/update entry in SITE_CHANGELOG array in app.js
  • Increment ?v=N cache params on CSS/JS script tags

2. Token-First CSS

uds-token-first-css.mdc
  • No hardcoded hex colors, rgb(), or rgba() — use var(--uds-color-*)
  • No hardcoded px for font-size, spacing, radius — use var(--uds-*) tokens
  • No hardcoded font-family — use var(--uds-font-family)
  • Shadows use var(--uds-shadow-*) tokens

3. Component Checklist

uds-component-checklist.mdc
  • CSS: :hover, :focus-visible, disabled states
  • JS (when interactive): IIFE pattern, _udsInit guard, keyboard handlers
  • Registered in uds.js orchestrator
  • PLAYGROUNDS, IMPL_DATA, COMPONENT_STATUS, FIGMA_LINKS entries in app.js
  • CHANGELOG entry under category: "components"
  • CSS + JS files added to download ZIP array

4. Content Schema (spec source of truth)

uds-content-schema.mdc
  • Per-component spec lives in content/<component>.json
  • Edit the JSON, NEVER re-introduce spec HTML into index.html
  • Empty fields are auto-hidden by the renderer — no placeholder text needed
  • Required minimum: component, title, description, whenToUse, dependencies, accessibility.keyboard, owner
  • Aim for 80%+ spec completeness before marking Production Ready

5. Release Workflow

uds-release-workflow.mdc
  • If UDS CHANGELOG was modified, sync Figma release notes IMMEDIATELY
  • Rebuild Release Notes in both UDS Tokens and UDS Components files
  • Never defer — do it before completing the task

Button

A button triggers an event or action. Switch the Brand control to see accent color changes.

Variants

Primary, secondary, and ghost buttons provide different levels of visual emphasis.

Show code
<button class="udc-button-primary">Primary</button>
<button class="udc-button-secondary">Secondary</button>
<button class="udc-button-ghost">Ghost</button>

Sizes

Small and default sizes. Use data-size="sm" for compact variant.

Show code
<button class="udc-button-primary" data-size="sm">Small</button>
<button class="udc-button-primary">Default</button>

Leading icon

Add a leading icon with data-leading-icon. Left padding is automatically reduced.

Show code
<button class="udc-button-primary" data-leading-icon>
  <span class="material-symbols-outlined">add</span> Create
</button>

Trailing icon

Add a trailing icon with data-trailing-icon. Right padding is automatically reduced.

Show code
<button class="udc-button-primary" data-trailing-icon>
  Next <span class="material-symbols-outlined">arrow_forward</span>
</button>

Icon only

Square buttons with only an icon. Use data-icon-only and always provide an aria-label.

Show code
<button class="udc-button-primary" data-icon-only aria-label="Add">
  <span class="material-symbols-outlined">add</span>
</button>

Destructive

Use data-btn-color="danger" on the button or a parent to signal destructive actions (maps to Figma's Destructive=True).

Show code
<div data-btn-color="danger">
  <button class="udc-button-primary" data-leading-icon>
    <span class="material-symbols-outlined">delete</span> Delete
  </button>
</div>

Disabled

Disabled buttons use a dedicated fill and text color — not opacity.

Show code
<button class="udc-button-primary" disabled>Disabled</button>

Selected

Toggle-style selection via aria-selected="true".

Show code
<button class="udc-button-primary" aria-selected="true">Selected</button>
