Migration Guides

Per-version upgrade notes. Read these when bumping a project from one UDS version to another. Breaking changes are flagged.

0.1 → 0.2

No breaking changes for token consumers. The doc site has been reshaped into a design-to-engineering handoff spec; framework-specific code (React/Vue) was removed from the docs and now lives in Storybook.

For developers

  • If you were using the React/Vue snippets from the doc site as production code, switch to the Storybook web components instead.
  • If you were importing uds-core/uds.css for tokens, no change needed — that path is stable.
  • The downloadable Cursor rule (uds-design-system.mdc) was simplified to vanilla-only patterns. Re-download to refresh.

For designers

  • Component pages now have a Guidelines tab (renamed from Usage) backed by per-component JSON spec files. Edit those instead of the HTML directly.
  • Spec completeness is now tracked separately from production status — see the colored dot in the sidebar.

Future versions

Migration notes for future UDS versions will appear here as releases ship.
