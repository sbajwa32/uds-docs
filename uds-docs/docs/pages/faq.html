FAQ

Common questions about working with UDS.

How do I switch themes?
Set data-color-scheme, data-theme, data-font, data-font-scale, or data-density on the <html> element. Semantic tokens automatically resolve to the correct values. Your component code never changes — only the attribute on the root element does.
How do I add a new token?
Tokens are sourced from Figma. To request a new token, propose it in the Figma file first, get it approved, then export the updated JSON. The release workflow rule will guide you through importing into uds/tokens/.
How do I request a new component?
Add an entry to the Roadmap page (proposed status). Once approved, follow the How to Contribute guide.
How do I report a bug or design issue?
Click the Report Issue button on any component page — it opens a GitHub issue prefilled with that component's context. For general site bugs, file an issue at the repo.
How does versioning work?
Two versions exist: UDS X.Y (the design system itself, bumped when tokens or components change) and SITE YYYY.MM.DD.N (this documentation site, bumped on every site change). Use the version dropdown in the header to view older snapshots.
Why is all the code "Example only"?
The doc site is the design specification. Production components live in Storybook as framework-agnostic web components. Code on this site is a reference for what the spec implies — it's not what you ship.
What happens to the spec when a component is updated?
Update content/<component>.json first (the source of truth for the spec). The Guidelines tab and AI context regenerate automatically. Then update the visual implementation in Storybook to match.
How do AI agents consume UDS?
Two ways: (1) point them at /ai-assist on this site, or (2) drop the downloadable uds-design-system.mdc Cursor rule into .cursor/rules/ in any project. The rule is self-contained — no internet fetch required.
Why are motion and responsive sections empty?
UDS doesn't yet have motion tokens or breakpoint tokens. Until those land in the foundation layer, those Guidelines sections are intentionally deferred to avoid arbitrary values polluting the spec. They're tracked in the Roadmap.
What's the difference between Composition Recipes and the Demo Builder?
Composition Recipes are curated patterns — vetted multi-component layouts you should reach for. Demo Builder is an experimentation sandbox — pick any components and mock something up. Use Recipes for handoff references; use Demo Builder for ad-hoc exploration.
