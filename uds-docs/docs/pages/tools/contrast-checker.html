Contrast Checker

Live audit of every --uds-color-text-*, --uds-color-icon-*, and --uds-color-border-* foreground token against every --uds-color-surface-* background. Contrast is computed client-side from the currently-resolved CSS variables, so flipping the theme bar above (light/dark, brand, font scale, density) recomputes every cell in place. Use this to validate token pairings across all six supported themes without leaving the page.

WCAG thresholds

The page uses kind-aware thresholds so verdicts match the standards that actually apply.

  • Text — WCAG SC 1.4.3: 4.5:1 for normal body text (AA), 7:1 for AAA, and 3:1 for large text (18 pt+ regular or 14 pt+ bold).
  • Icons and borders — WCAG SC 1.4.11 Non-text Contrast: 3:1 against the adjacent surface. SC 1.4.11 doesn't define an AAA tier.
  • Transparent tokens (*-none) are skipped — contrast is undefined against / using a transparent fill. Tokens whose name contains -disabled are intentionally low-contrast by design; the ratio is still shown, but the verdict pill is labelled "by design" so the result is not misread as a bug.
